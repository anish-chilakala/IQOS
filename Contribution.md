## Contributing

We highly encourage contribution to this project. If you choose to contribute, you can contact me at anish.chilakala@gmail.com

## Contribution Guidelines

If you want to contribute, please follow these guidelines:

- All code must be written in C++ and follow the project's coding style
- Every new module must include its own documentation
- Every new feature must include unit tests before submission
- Do not break backwards compatibility without a major version bump
- All pull requests must include a clear description of what was changed and why
- All pull requests must be tested on real VEX IQ hardware before submission
- Do not submit code that depends on external libraries not already used in the project
- Keep modules self-contained — one module should not depend on another unless absolutely necessary
- Follow the existing folder and file naming conventions
- All contributions must respect VEX's Student Centered Policy and competition rules
- If you find a bug, open an issue before submitting a fix so it can be tracked
- If you want to add a major feature, open an issue for discussion before writing code
- Credit yourself in the file header of any file you create or significantly modify
